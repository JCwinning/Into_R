4 Basic Structure of a Shiny App

A Shiny app consists of two main components:

4.1 1. User Interface (UI)

The UI defines how your app looks and includes: - Input controls (sliders, dropdowns, text boxes) - Output displays (plots, tables, text) - Layout and styling

4.2 2. Server Logic

The server function: - Processes user inputs - Performs calculations and data manipulation - Generates outputs to display in the UI

5 Basic Shiny App Structure

Code 
library(shiny)

# Define UI
ui <- fluidPage(
  titlePanel("My First Shiny App"),

  sidebarLayout(
    sidebarPanel(
      sliderInput("num", "Choose a number:",
                  min = 0, max = 100, value = 50)
    ),

    mainPanel(
      textOutput("result")
    )
  )
)

# Define server logic
server <- function(input, output) {
  output$result <- renderText({
    paste("You selected:", input$num)
  })
}

# Run the application
shinyApp(ui = ui, server = server)

6 Key Shiny Components

6.1 Input Widgets

Common input types include: - sliderInput(): Numeric sliders - textInput(): Text input fields - selectInput(): Dropdown menus - checkboxInput(): Checkboxes - radioButtons(): Radio button groups

6.2 Output Functions

Common output types include: - textOutput(): Display text - plotOutput(): Display plots - tableOutput(): Display tables - uiOutput(): Display dynamic UI elements

6.3 Reactive Expressions

Reactive expressions automatically update when their inputs change:

Code 
# Create a reactive expression
reactive_data <- reactive({
  # This code runs whenever its dependencies change
  input$dataset %>%
    filter(category == input$category)
})

# Use the reactive expression in output
output$plot <- renderPlot({
  ggplot(reactive_data(), aes(x, y)) +
    geom_point()
})

# Basic Structure of a Shiny App

A Shiny app consists of two main components:

## 1. User Interface (UI)

The UI defines how your app looks and includes:
- Input controls (sliders, dropdowns, text boxes)
- Output displays (plots, tables, text)
- Layout and styling

## 2. Server Logic

The server function:
- Processes user inputs
- Performs calculations and data manipulation
- Generates outputs to display in the UI

# Basic Shiny App Structure

```{r}
#| eval: false
library(shiny)

# Define UI
ui <- fluidPage(
  titlePanel("My First Shiny App"),

  sidebarLayout(
    sidebarPanel(
      sliderInput("num", "Choose a number:",
                  min = 0, max = 100, value = 50)
    ),

    mainPanel(
      textOutput("result")
    )
  )
)

# Define server logic
server <- function(input, output) {
  output$result <- renderText({
    paste("You selected:", input$num)
  })
}

# Run the application
shinyApp(ui = ui, server = server)
```
